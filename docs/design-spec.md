# EasyZ 官网重构 — 设计 Spec

> 状态：v0.2 草稿（持续更新中）
> 维护人：Mavis + 用户
> 最近更新：2026-07-24

---

## 0. 项目背景

EasyZ（`easyzai.top`）是「**研究型 AI 落地团队**」，专注把 AI 落进复杂、非标的真实业务场景。区别于普通外包：做的是「问题研究 + 方案设计 + 工程交付」全链路，团队小、研究背景、专门啃硬骨头。

官网是团队对外的第一门面，需要传达「**研究底色 + 工程能力**」，而不是「又一个 AI 公司」模板感。

---

## 1. 当前现状（问题盘点）

| 维度 | 现状 | 问题 |
|---|---|---|
| 首页结构 | 9 个 section 堆在一页（Hero / TrustBar / Services / Cases / LiveDemo / WhyUs / Partners / CTA / Footer） | 太长，信息密度过高 |
| 文字 | 大量描述性段落 | 没人爱看 |
| 视觉 | UI 太素，无图 | 缺乏品牌质感 |
| NavBar | 6 个标签，其中 5 个是首页锚点，只有 `thinking` 有真路由 | 结构与导航不一致 |
| Hero | Ferrofluid WebGL 动画（ogl） | 「AI 公司模板」感重 |
| 主题 | 浅 / 深双主题 | 新设计不沿用 |
| 部署 | `output: 'export'` 静态导出 | 保留 |
| i18n | 中英双语（next-intl） | **保留** |

---

## 2. 重构目标

1. **多页结构**：4 个主页面（首页 / 关于 / 实践 / 思考），每个 NavBar 标签对应独立路由
2. **首页只做品牌门面**，不堆 section
3. **全站加入「概念图」** —— 通用视觉资产，散落各页
4. **整体克制、视觉化** —— 减少文字密度
5. **首页 Hero 用概念视频** 作背景
6. **保留中英双语**

## 3. 视觉语言

### 3.1 颜色（建议初值，待用户拍板）

| 角色 | 色值 | 说明 |
|---|---|---|
| 主色 建筑白 | `#F4F2EE` | 暖白，不刺眼 |
| 背景白 | `#FAF8F4` | 比主色更亮一档 |
| 点缀 蓝晒蓝 | `#1B3A5F` | 深蓝，不抢戏；**只作为点缀** |
| 大理石（材质） | 灰白 + 自然纹路 | 仅用于「关于」「实践」两页 |
| 文字 primary | `#1A1A1A` | 接近黑 |
| 文字 secondary | `#5A5A5A` | 灰 |
| 文字 tertiary | `#9A9A9A` | 浅灰 |
| 边框 subtle | `#E5E2DC` | 极淡 |

**色彩使用原则**：
- 白铺底，蓝出现 1 次算点缀
- 同一页面内，**不允许两处都用主色之外的强色**（避免层级垮塌）
- 大理石只出现在「关于」「实践」

### 3.2 字体

| 角色 | 字体 | 用途 |
|---|---|---|
| Sans | Inter（现有，next/font） | 正文 / UI |
| Serif | **新增**：Source Serif Pro 或 Newsreader | 概念图 caption / 长文 |
| Mono | JetBrains Mono（现有，next/font） | 日期 / 标签 / 元数据 |

### 3.3 字号 / 行高

```
12 / 14 / 16 / 20 / 32 / 56
```

- 正文行高：1.5
- Display 行高：1.1

### 3.4 间距 / 容器

- 间距系统：`8 / 16 / 24 / 48 / 96 / 160`
- 容器 max-w：`1280px`（页面） / `640px`（正文阅读）

### 3.5 动效

- 缓动：`cubic-bezier(0.25, 0.46, 0.45, 0.94)`（与现有相同）
- 入场：文字 0.6s / 区块 0.8s
- 概念图视差：±10px 范围，10s+ 一周期自动循环

### 3.6 视觉气质

> **「把研究当手艺，不当表演」**

参考意象：
- 蓝晒植物标本（cyanotype）—— 19 世纪科学摄影法，研究底色
- 建筑仰拍（白空间 + 光影）—— 留白是内容，不表演

## 5. 首页 Spec

### 5.1 节奏

```
[ Hero 视频     100vh       ]
[ 定位段         60vh 留白主导 ]
[ 概念图         80-100vh 静默 ]
[ Recent Thinking  50vh      ]
[ CTA           40vh        ]
[ Footer        auto        ]
```

**总高 ≈ 5–6 屏**。

**不上**：TrustBar / Partners / 演示 / FAQ / 客户 logo / 客户证言。

### 5.2 Hero

**视觉**：
- 概念视频**全屏铺底**（100vw × 100vh），loop，无音
- 方向 b —— **建筑 / 空间 / 光影慢镜头**（无品牌色滤镜，纯白 + 自然光）
- 时长 12–20s，体积 < 4MB（首屏 LCP 考虑）

**文字层**（视频上方叠）：
- 不放品牌标语
- 3 件东西：
  - 品牌 mark（极小，≤ 24px 高）
  - tagline 1 行（≤ 16px）
  - 向下滚动细线箭头（居中底部）
- 全部在右下角小区域 或 居中极小

**暗色兜底**：视频加载失败时回退到 1 张静态建筑白图 + 同款文字层。

### 5.3 定位段

- 容器：`max-w-[640px]`，垂直居中
- 内容（2 行）：
  ```
  我们研究 AI 落地，专啃非标复杂场景。
  问题研究 · 方案设计 · 工程交付
  ```
- 第一行 28–32px，第二行 14–16px
- 行高 1.4–1.5
- **无 bullet、无图标、无按钮**

### 5.4 概念图 section

- 1 张图：蓝晒植物标本（参考用户提供的样图）
- 满铺或居中
- **0 文字**
- 微动效：鼠标 hover 视差 ±10px，10s+ 一周期自动循环
- 主体悬浮感：四周大量留白，不靠边

> **关键决定（2026-07-23 已锁）**：Hero 走白色（方向 b），概念图走蓝晒蓝。两者**换语言**，蓝晒蓝作为全站 1 个标志性的颜色 moment。
>
> **资产已锁（2026-07-24）**：见 `docs/home-assets.md` —— Hero 视频 = `hero-4.mp4`（大理石墙面 dolly，6s），概念图 = `concept-cyanotype-3.png`（蓝晒 sparse branch）。

### 5.5 Recent Thinking

- 3 篇，左对齐，**不做卡片**
- 每行结构：
  ```
  2026.07.18  从一个券商案例看 AI 落地的"非标"边界
              —— 1 行摘要 ≤ 60 字                继续阅读 →
  ```
- 日期用 mono 字体，13–14px
- 标题 18–20px，hover 颜色变蓝晒蓝
- 整行可点
- 底部 1 行：`查看全部思考 →` 跳 `/thinking`

### 5.6 CTA / 收尾

- "开始沟通" 细描边按钮 + 邮箱地址
- 居中或左对齐
- 文字层级 14–16px
- 不放"立即咨询 / 免费试用 / 获取报价"

### 5.7 Footer

- 重复一次 CTA 入口
- 中英切换
- 主题切换（如果保留）
- 版权信息

## 6. 其他三页（概要）

### 6.1 关于 `/[locale]/about`

按"研究报告"语气写，不是公司介绍页：

1. **开场**：一段短叙述（≤ 80 字）说明 EasyZ 是什么、为什么存在
2. **团队**：成员照片 + 名字 + 角色 + 1 行简介
3. **原则**：3–5 条「我们怎么做事」（每条 ≤ 20 字）
4. **石材收尾**：1 个大理石 section

### 6.2 实践 `/[locale]/practice`（最重的一页）

scroll 阅读，不分页，3 段：

1. **方法论** —— 我们怎么解题（研究 → 方案 → 交付 三段式，每段标题 + 2–3 句）
2. **能力图谱** —— 概念图 / 抽象视觉代替 live demo
   - chat / outreach / code-migration 等能力用静态视觉表达，**不再做交互 demo**
3. **案例** —— 当前已有 2 个 case（programming-language-migration / securities-ai-platform）
   - 走现有 `/cases/[slug]` 路由
   - 留位给未来 2–4 个

### 6.3 思考 `/[locale]/thinking`（保持现状）

- 列表：日期 / 标题 / 1 行摘要 / 标签
- 详情：长文阅读，宽松行距，收窄最大宽，1 张概念图随文
- 整体改成"出版物"风
