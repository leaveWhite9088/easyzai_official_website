# EasyZ 官网重构 — 设计 Spec

> 状态：v0.2 草稿（持续更新中）
> 维护人：Mavis + 用户
> 最近更新：2026-07-24

---

## 0. 项目背景

EasyZ（`easyzai.top`）是「**研究型 AI 落地团队**」，专注把 AI 落进复杂、非标的真实业务场景。区别于普通外包：做的是「问题研究 + 方案设计 + 工程交付」全链路，团队小、研究背景、专门啃硬骨头。

官网是团队对外的第一门面，需要传达「**研究底色 + 工程能力**」，而不是「又一个 AI 公司」模板感。

## 1. 当前现状（问题盘点）

| 维度 | 现状 | 问题 |
|---|---|---|
| 首页结构 | 9 个 section 堆在一页（Hero / TrustBar / Services / Cases / LiveDemo / WhyUs / Partners / CTA / Footer） | 太长，信息密度过高 |
| 文字 | 大量描述性段落 | 没人爱看 |
| 视觉 | UI 太素，无图 | 缺乏品牌质感 |
| NavBar | 6 个标签，其中 5 个是首页锚点，只有 `thinking` 有真路由 | 结构与导航不一致 |
| Hero | Ferrofluid WebGL 动画（ogl） | 「AI 公司模板」感重 |
| 主题 | 浅 / 深双主题 | 新设计不沿用 |
| 部署 | `output: 'export'` 静态导出 | 保留 |
| i18n | 中英双语（next-intl） | **保留** |

## 2. 重构目标

1. **多页结构**：4 个主页面（首页 / 关于 / 实践 / 思考），每个 NavBar 标签对应独立路由
2. **首页只做品牌门面**，不堆 section
3. **全站加入「概念图」** —— 通用视觉资产，散落各页
4. **整体克制、视觉化** —— 减少文字密度
5. **首页 Hero 用概念视频** 作背景
6. **保留中英双语**

## 3. 视觉语言

### 3.1 颜色（建议初值，待用户拍板）

| 角色 | 色值 | 说明 |
|---|---|---|
| 主色 建筑白 | `#F4F2EE` | 暖白，不刺眼 |
| 背景白 | `#FAF8F4` | 比主色更亮一档 |
| 点缀 蓝晒蓝 | `#1B3A5F` | 深蓝，不抢戏；**只作为点缀** |
| 大理石（材质） | 灰白 + 自然纹路 | 仅用于「关于」「实践」两页 |
| 文字 primary | `#1A1A1A` | 接近黑 |
| 文字 secondary | `#5A5A5A` | 灰 |
| 文字 tertiary | `#9A9A9A` | 浅灰 |
| 边框 subtle | `#E5E2DC` | 极淡 |

**色彩使用原则**：
- 白铺底，蓝出现 1 次算点缀
- 同一页面内，**不允许两处都用主色之外的强色**（避免层级垮塌）
- 大理石只出现在「关于」「实践」

### 3.2 字体

| 角色 | 字体 | 用途 |
|---|---|---|
| Sans | Inter（现有，next/font） | 正文 / UI |
| Serif | **新增**：Source Serif Pro 或 Newsreader | 概念图 caption / 长文 |
| Mono | JetBrains Mono（现有，next/font） | 日期 / 标签 / 元数据 |

### 3.3 字号 / 行高

```
12 / 14 / 16 / 20 / 32 / 56
```

- 正文行高：1.5
- Display 行高：1.1

### 3.4 间距 / 容器

- 间距系统：`8 / 16 / 24 / 48 / 96 / 160`
- 容器 max-w：`1280px`（页面） / `640px`（正文阅读）

### 3.5 动效

- 缓动：`cubic-bezier(0.25, 0.46, 0.45, 0.94)`（与现有相同）
- 入场：文字 0.6s / 区块 0.8s
- 概念图视差：±10px 范围，10s+ 一周期自动循环

### 3.6 视觉气质

> **「把研究当手艺，不当表演」**

参考意象：
- 蓝晒植物标本（cyanotype）—— 19 世纪科学摄影法，研究底色
- 建筑仰拍（白空间 + 光影）—— 留白是内容，不表演

## 5. 首页 Spec

### 5.1 节奏

```
[ Hero 视频     100vh       ]
[ 定位段         60vh 留白主导 ]
[ 概念图         80-100vh 静默 ]
[ Recent Thinking  50vh      ]
[ CTA           40vh        ]
[ Footer        auto        ]
```

**总高 ≈ 5–6 屏**。

**不上**：TrustBar / Partners / 演示 / FAQ / 客户 logo / 客户证言。

### 5.2 Hero

**视觉**：
- 概念视频**全屏铺底**（100vw × 100vh），loop，无音
- 方向 b —— **建筑 / 空间 / 光影慢镜头**（无品牌色滤镜，纯白 + 自然光）
- 时长 12–20s，体积 < 4MB（首屏 LCP 考虑）

**文字层**（视频上方叠）：
- 不放品牌标语
- 3 件东西：
  - 品牌 mark（极小，≤ 24px 高）
  - tagline 1 行（≤ 16px）
  - 向下滚动细线箭头（居中底部）
- 全部在右下角小区域 或 居中极小

**暗色兜底**：视频加载失败时回退到 1 张静态建筑白图 + 同款文字层。

### 5.3 定位段

- 容器：`max-w-[640px]`，垂直居中
- 内容（2 行）：
  ```
  我们研究 AI 落地，专啃非标复杂场景。
  问题研究 · 方案设计 · 工程交付
  ```
- 第一行 28–32px，第二行 14–16px
- 行高 1.4–1.5
- **无 bullet、无图标、无按钮**

### 5.4 概念图 section

- 1 张图：蓝晒植物标本（参考用户提供的样图）
- 满铺或居中
- **0 文字**
- 微动效：鼠标 hover 视差 ±10px，10s+ 一周期自动循环
- 主体悬浮感：四周大量留白，不靠边

> **关键决定（2026-07-23 已锁）**：Hero 走白色（方向 b），概念图走蓝晒蓝。两者**换语言**，蓝晒蓝作为全站 1 个标志性的颜色 moment。
>
> **资产已锁（2026-07-24）**：见 `docs/prototype-assets.md` —— Hero 视频 = `hero-4.mp4`（大理石墙面 dolly，6s），概念图 = `concept-cyanotype-3.png`（蓝晒 sparse branch）。

### 5.5 Recent Thinking

- 3 篇，左对齐，**不做卡片**
- 每行结构：
  ```
  2026.07.18  从一个券商案例看 AI 落地的"非标"边界
              —— 1 行摘要 ≤ 60 字                继续阅读 →
  ```
- 日期用 mono 字体，13–14px
- 标题 18–20px，hover 颜色变蓝晒蓝
- 整行可点
- 底部 1 行：`查看全部思考 →` 跳 `/thinking`

### 5.6 CTA / 收尾

- "开始沟通" 细描边按钮 + 邮箱地址
- 居中或左对齐
- 文字层级 14–16px
- 不放"立即咨询 / 免费试用 / 获取报价"

### 5.7 Footer

- 重复一次 CTA 入口
- 中英切换
- 主题切换（如果保留）
- 版权信息

## 6. 其他三页（概要）

### 6.1 关于 `/[locale]/about`（已重新切分）

**角色定位**：About = 「**我们是谁 + 我们相信什么**」= 哲学层（why）。最轻的一页（3 屏内）。**不放团队人像**——避免"咨询公司 / 设计工作室"那种"展示我们是谁"的高端感，转向"展示我们相信什么"。

**4 块结构**：

1. **`01 / 段头`** — 1 段开场叙述（≤ 80 字）说 EasyZ 是什么、为什么存在、**为谁解决什么问题**
   - **唯一使用 serif 的位置**——这一行"压住"页面
2. **`02 / 原则`** — 3–5 条「我们怎么做事」（每条 ≤ 20 字）
   - 用 Inter / mono，**不上 serif**
   - 例：
     - "先研究，再动手"
     - "做不到就说做不到"
     - "代码能跑的才算交付"
3. **`03 / 我们如何工作（高层）`** — 1–2 段叙述，不展开细节（细节去 Practice）
   - 起到"导引"作用：读完这段，自然想点 Practice 看具体怎么做的
   - **大理石纹理**作为这一段的背景材质（极淡、不抢戏）
4. **`04 / 同路人`** — 合作伙伴 logos 横向排列
   - 替换原 CTA（首页 / Footer 已有 CTA，About 再放就重复了）
   - 当前有 3 家真实合作伙伴：**云汉孵化中心 / 智猩猩 / 连尚集团**
   - 视觉处理：灰度（`grayscale(100%)`）+ 轻微 `opacity: 0.7` 压住
   - framing：**"与我们一起探索 AI 落地的同路人"**——不是"客户"姿态，是"共同探索"姿态
   - 不要放通用"我们与 XX 合作"的话术

**视觉降级原则**：
- section 编号保留（`01 /` `02 /` `03 /` `04 /` 编辑式痕迹保留"这页有结构"）
- **serif 只用在第 01 段开场**——一处"压住"即可
- 其他标题、正文、按钮全部 Inter / mono

**与 Practice 的边界**：
- About = why（哲学 / 信念 / 同路人）
- Practice = what / how / done（方法论 / 能力 / 案例）
- 两者**绝不重复**

### 6.2 实践 `/[locale]/practice`（最重的一页 · practitioner first）

**角色定位**：Practice = 「**我们做什么 + 做过什么**」= 实战层（what / done）。**不复述方法论**——"我们怎么工作"在 About 03 段已经写过；Practice 的重心是**案例**（主菜），能力图谱是支撑。

**3 段结构**：

1. **`01 / 能力图谱`** — 4 项核心能力（2×2 网格，每项标题 + 1 行描述）
   - 大理石纹理做背景（材质支撑）
   - **不展开方法论**——只看"我们能做什么"
   - 4 项：
     - AI 代码迁移与知识工程
     - 企业系统 AI 化改造
     - RAG / Agent 系统搭建
     - 非标场景的端到端交付
2. **`02 / 案例`** — **主菜**。4 个 case 摘要行（紧凑列表式，每行 ~120px）
   - 数据来源：`src/data/cases-meta.ts` + `messages/{zh,en}.json` 的 `cases.items`
   - 4 个 case：
     - **CASE 01 · 编程语言生态库迁移工具**（科技 / 开发者工具，80%+ 库迁移自动化）—— 有 detail 页
     - **CASE 02 · 证券中台管理系统 AI 化改造**（金融科技，70%+ 审批周期缩短）—— 有 detail 页
     - **CASE 03 · 激光加工设备企业智能客服**（制造业，75%+ 常见问题自动解决率）—— 暂无 detail 页
     - **CASE 04 · 出海企业海外获客自动化系统**（外贸 / 跨境电商，3x+ 客户开发效率提升）—— 暂无 detail 页
   - 行结构（**12-col grid**，全部 baseline 对齐）：
     - 左 2 栏：CASE 编号（mono uppercase）+ metric（serif 28px）+ 行业（mono）
     - 中 9 栏：标题（sans 19px）+ 1 行摘要（ink-2 14px）
     - 右 1 栏：→ 箭头
   - 行间用 hairline 分隔（border-top border-rule）
   - 整行可点 → 现有 `/cases/[slug]` 路由（CASE 01, 02）；CASE 03, 04 暂时链 `#`，等 detail 页实现
   - hover：背景极淡 paper + 标题/metric 变蓝晒蓝 + 箭头右移
   - **不使用**大理石缩略图（之前的卡片式太"重"，改为紧凑列表）
3. **`03 / 收尾`** — 1 句提问 + CTA
   - 标题：「**你手上有什么啃不动的非标？**」（serif）
   - 1 句补充：「写信告诉我们。研究、方案、交付，三段全做。」
   - CTA：「开始沟通」描边按钮 + 邮箱

**视觉语言**：
- section 编号保留（`01 /` `02 /` `03 /`）
- serif 用在 01 段开场（"啃过的东西，列在这里。"）和 03 段提问（"你手上有什么啃不动的非标？"）—— 2 处"压住"
- 案例卡的缩略图先用大理石纹理作占位，metric 大号用 serif（80%+ / 70%+）撑视觉
- 整体降级与 About 一致

**与 About 的边界**（关键不重叠）：
- About 03 = "**我们怎么工作**"（how）—— 高层 1-2 段，导引至 Practice
- Practice 01-02 = "**我们能做什么 / 做过什么**"（what / done）—— 案例 + 能力
- 两页**绝不重复**

### 6.3 思考 `/[locale]/thinking`（出版物风）

**角色定位**：Thinking = 「**我们想清楚的事，写下来**」= 阅读层（essays / notes）。3 类文章：**方法论 / 案例复盘 / 技术观点**（沿用现有数据）。3 篇实际文章已在 `messages/{zh,en}.json` 的 `thinking.articles[]` 中。

**2 个页面原型**：

#### 6.3.1 列表页 `/[locale]/thinking`（`docs/prototype/thinking.html`）

- 顶部：**1 句话 intro**（pt-24, pb-8）—— 「想清楚的事，写下来。」
  - **不**做成模块，**不**加 section 编号 / 大标题 / 副标题 —— 最简
- 下面直接是文章列表（02 / archive 段，section 编号保留）：
  - 每篇：编号（mono 01-03）+ 日期（mono）+ 分类（cat-tag 描边小标）+ 标题（sans 20-24px）+ 摘要（ink-2 14-15px）+ 「阅读全文 →」
  - 行间用 hairline 分隔，hover 背景 paper + 标题变蓝晒蓝 + 箭头右移
  - 底部：「3 articles · 持续更新」+ 「RSS feed →」（占位）

#### 6.3.2 详情页 `/[locale]/thinking/[slug]`（`docs/prototype/thinking-detail.html`）

- **顶部概念图**：**50vh 高 / 100vw 全宽 / `object-cover`**（fixed nav 之下第一个 section）
  - 横向铺满屏幕，纵向占满半个屏幕
  - 下拉就是文章（图文分离，无 overlay）
- 文章区（`pt-12 / pb-24`）：
  - 「← 思考」返回链接（mono 11px）
  - Header：编号 + 日期 + 分类（mono）+ 标题（serif 28-44px，唯一一处 serif）
  - 正文（max-w-[820px]）：markdown 渲染
    - 段落 16px / line-height 1.75
    - H2：serif 26px（阅读页"出版物"特征）
    - 列表、引用、代码块都有专门样式
    - 块引用左侧蓝晒蓝边
- 收尾：上一篇 / 下一篇（左右分栏）

**概念图 → 文章调性映射**（与列表页 spec 一致）：
- 方法论类 → `concept-cyanotype-3.png`（蓝晒植物标本，research / specimen 感）
- 案例复盘类 → `marble-texture.png`（大理石，material / work 感）
- 技术观点类 → `thinking-arch.png`（建筑结构，structural 感）

**视觉语言**：
- section 编号在列表页保留（01 / 02），详情页用文章编号（02，对应"第 2 篇"）
- **不使用大理石**（spec 规定 marble 只在 About + Practice）
- 列表页和详情页都保持克制——文字为主，概念图作为视觉锚点
- 详情页 H2 用 serif（出版物特征），不是"过度使用"——这页是阅读页，serif 是"读"的语言

**3 篇文章 → 概念图映射**：

| # | 标题 | 分类 | 概念图 |
|---|---|---|---|
| 01 | 在接 AI 项目前，我们用 5 个问题决定要不要接 | 方法论 | `concept-cyanotype-3.png` |
| 02 | 非侵入式改造一个证券中台 | 案例复盘 | `marble-texture.png` |
| 03 | 为什么我们不无脑用 Dify / LangChain | 技术观点 | `thinking-arch.png` |
